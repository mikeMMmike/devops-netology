Задача 1
=====

* **Опишите своими словами основные преимущества применения на практике IaaC паттернов.**

**Ответ.**
```
Основные приемущества: 
1. Снижение цены устранения ошибки в разработке путем непрерывного автоматизированного слияния веток разработки и тестирования внесенных изменений. 
2. Уменьшение рутинных задач за счет автоматизации процессов по слиянию веток и проведения тестов
3. Уменьшение финансовых и времязатрат на утверждение кода (в ветках для разработчиков). 
т.е. основная задача - максимально автоматизировать процесс разработки, где это возможно без негативных последствий для бизнеса. 
```
* **Какой из принципов IaaC является основополагающим.**

**Ответ.**
```
На мой взгляд основополагающим принципом IaaC является устранение дрейфа конфигураций. т.е. главное - настроить инфраструктуру
таким образом, чтобы работа системы была предсказуемой, изменния конфигураций носили прозрачный характер, чтобы в инфраструктуре
не было места несовпадению сред разработки, тестирования и развертывания. При соблюдении этого принципа разработка будет эффективной, 
программный продукт будет быстрее производиться и доставляться до потребителя.
```

Задача 2
======
* **Чем Ansible выгодно отличается от других систем управление конфигурациями?**

**Ответ.**
```
Главная особенность, отличающая Ansible от других систем управление конфигурациями состоит в том,
что он использует существующую SSH инфраструктуру. Другие системы управления конфигурациями требуют
установки PKI-окружения.
```

* **Какой, на ваш взгляд, метод работы систем конфигурации более надёжный push или pull?**

**Ответ.**
```
На мой взгляд более надежным методом работы систем конфигурации является push, т.к. в этом случае
конфигурацию отправляет управляющий сервер. это позволяет как-то более централизованно что ли управлять
конфигурациеями серверов, а также инициировать отправку конфигурации вне расписания из одного места
```
