1.Какой системный вызов делает команда cd? В прошлом ДЗ мы выяснили, что cd не является самостоятельной программой, это shell builtin, поэтому запустить strace непосредственно на cd не получится. Тем не менее, вы можете запустить strace на /bin/bash -c 'cd /tmp'. В этом случае вы увидите полный список системных вызовов, которые делает сам bash при старте. Вам нужно найти тот единственный, который относится именно к cd. Обратите внимание, что strace выдаёт результат своей работы в поток stderr, а не в stdout.

strace /bin/bash -c 'cd /tmp' 2> strace-cd
grep "/tmp" strace-cd поведает о вызовах, в которых фигурирует параметр, переданный cd. разберем результаты:
execve("/bin/bash", ["/bin/bash", "-c", "cd /tmp"], 0x7ffdc31baa60 /* 27 vars */) = 0 - возвращает 0 после выполнения команды strace /bin/bash -c 'cd /tmp'
stat("/tmp", {st_mode=S_IFDIR|S_ISVTX|0777, st_size=4096, ...}) = 0 - Системный вызов stat возвращает информацию об указанном файле (и каталоге)
chdir("/tmp") - изменяет текущий каталог каталог на каталог из параметра, в нашем случае "/tmp"

Ответ: chdir - системный вызов, который делает команда cd

4. Занимают ли зомби-процессы какие-то ресурсы в ОС (CPU, RAM, IO)?
Зомби процессы не используют ресурсы и являются записью в таблице процессов для сохранения возможности рожительскому процессу прочитать вывод дочернего процесса-зомби. Зомби процесс может навредить лишь переполнением таблицы процессов, если зомби процессов много в системе. но это больше исключение нежели правило. зомби процессы завершаются при завершении родительского процесса.

7. Чем отличается последовательность команд через ; и через && в bash? Например:
root@netology1:~# test -d /tmp/some_dir; echo Hi
Hi
root@netology1:~# test -d /tmp/some_dir && echo Hi
root@netology1:~#

Ответ: ; позволяет выполнить команды последовательно, независимо от результата работы первой. && запускает следующую команду только в том случае, если первая команда выполнена успешно.

Есть ли смысл использовать в bash &&, если применить set -e?
Смысла использовать && нет, т.к. при использовании set -e происходит немедленный выход, если команда завершается ненулевым статусом

8. Из каких опций состоит режим bash set -euxo pipefail и почему его хорошо было бы использовать в сценариях?

e - Немедленно выйти, если команда завершается с ненулевым статусом.
u - При подстановке расценивайте неустановленные переменные как ошибку.
x - Печатайте команды и их аргументы когда они выполнились.
o - имя-опции  pipefail:
возвращаемое значение конвейера - это статус последней команды для выхода с ненулевым статусом или ноль, если ни одна команда не вышла с ненулевым статусом

Почему это хорошо использовать  в скриптах:
потому что при использовании bash set -euxo pipefail скрипт будет выполнен, только если все переменные будут установлены, а также печатать команды по мере их выполнения (удобно котроллировать визуально, какую команду сейчас выполняет скрипт) и завершаться при возникновении ошибки в любой из команд, включая случай использования | если предыдущая команда отработала с ошибкой.
