Задача 1
===========
Опишите кратко, как вы поняли: в чем основное отличие полной (аппаратной) виртуализации, паравиртуализации и виртуализации на основе ОС.

Ответ.
-------------

Основное отличие типов виртуализации между собой, на мой взягляд, состоят в способе предоставления ресурсов для ВМ:
<li>
Полная аппаратная виртуализация предоставляет драйверы для доступа к ресурсам
<li>
Паравиртуализация использует ядро ОС для предоставления ресурсов ВМ
<li>
Виртуализация на основе ОС использует ядро ОС для запуска и предоставления ресурсов ВМ . в данном случае гостевые ВМ могут базироваться лишь на ядре хостовой ВМ.
 
Каждый тип виртуализации служит для реализации определенных целей и решения конкретных задач: 
под каждую задачу выбирается оптимальное решение с т.з. производительности, возможностей, бюджета и ресурсоемкости.


Задача 2
===========
Выберите один из вариантов использования организации физических серверов, в зависимости от условий использования.

Организация серверов:
<li>
физические сервера
<li>
паравиртуализация
<li>
виртуализация уровня ОС.

Условия использования:
<li>
Высоконагруженная база данных, чувствительная к отказу.
<li>
Различные web-приложения.
<li>
Windows системы для использования бухгалтерским отделом.
<li>
Системы, выполняющие высокопроизводительные расчеты на GPU.

Опишите, почему вы выбрали к каждому целевому использованию такую организацию.


Ответ.
-------------
Сортировку будем выполнять по условиям использования. итак:
<li>
Высоконагруженная база данных, чувствительная к отказу. Для этого условия прекрасно 
подойдет паравиртуализация, т.к. на этом типе виртуализации можно создать 
кластер гипервизоров с динамической миграцией в случае отказа 1 ВМ. Такое решение обеспечивает отказоустойчивость. 
Это существенно повысит надежность решения
<li>
Различные web-приложения.
Для web-приложений прекрасно подойдет виртуализация уровня ОС, т.к. различные web-приложения 
могут использовать одни и те же библиотеки и компоненты системы. данное решение позволит 
не усложняя инфраструктуру и не увеличивая стоимость обслуживания решить задачу.
<li>
Windows системы для использования бухгалтерским отделом.
Для этого условия подойдет паравиртуализация, т.к. она не накладывает ограничений 
на разворачиваемые гостевые ОС и позволит развернуть несколько инстансов серверов
на 1 или нескольких физических серверах. а также позволит отключить 
неиспользуемые ВМ, если в них пропадет необходимость. Физические сереры в данном случае 
бы простаивали, а виртуализация уровня ОС не смогла бы обеспечить широкий функционал
виртуальным машинам.
<li>
Системы, выполняющие высокопроизводительные расчеты на GPU.
Для решения данной задачи прекрасно подойдет старый-добрый физический сервер:)
т.к. именно в данном случае будут сведены к минимуму программные прослойки между 
приложением, ОС и оборудованием, что может является критичным показателем для задержек 
исполнения вычислений. т.е. в такой реализации приложение посредством ОС будет иметь
прямой доступ к оборудованию.

Задача 3
===========
Выберите подходящую систему управления виртуализацией для предложенного сценария. Детально опишите ваш выбор.

Сценарии:

1. 100 виртуальных машин на базе Linux и Windows, общие задачи, нет особых требований. Преимущественно Windows based инфраструктура, требуется реализация программных балансировщиков нагрузки, репликации данных и автоматизированного механизма создания резервных копий.
2. Требуется наиболее производительное бесплатное open source решение для виртуализации небольшой (20-30 серверов) инфраструктуры на базе Linux и Windows виртуальных машин.
3. Необходимо бесплатное, максимально совместимое и производительное решение для виртуализации Windows инфраструктуры.
4. Необходимо рабочее окружение для тестирования программного продукта на нескольких дистрибутивах Linux.

Ответ.
-------------
1. 100 ВМ - немалый парк. В этом случае подойдут как бесплатные продукты, так и платные: Hyper-v (включая бесплатный Hyper-V Sever), Xen PV: все зависит от деталей. 
В данном случае я бы остановил выбор на VMWare, т.к. VMWare предлагает решения по балансировке нагрузки, бекапу ВМ и репликации. 
2. Как бесплатное и высокопроизводительное решение для небольшого парка ОС
подойдет KVM с драйвером virtio. Скорость развертывания ВМ достигается за счет использования шаблонов ОС. 
драйвер virtio дает близкую к нативной производительность серверов. Решение имеет высокую совместимость с Linux м Windows системами. 
3. Бесплатное и максимально производительное решение для Windows-инфраструктуры - 
MS Windows Hyper-V Server. В данном режиме гипервизор является бесплатным, лицензировать его нет
необходимости. для win-инфраструктуры меет максимальную совместимость
4. Для организации рабочего окрудения дл тестирования на нескольких дистрибутивах ОС
прекрасно подойдет паравиртуализация в Virtual-Box в связке  с Vagrant. Vagrant отлично
интегрирутся с VirtualBox и при написании config-файла быстро развернет требуемые для 
тестирования ОС в гипервизоре VirtualBox. Решение можно применять прямо на ПК разработчика,
например без создания отдельного сервера. также сам разработчик может управлять ВМ, включая 
их когда есть необходимость.


Задача 4
===========
Опишите возможные проблемы и недостатки гетерогенной среды виртуализации (использования нескольких систем управления виртуализацией одновременно) и что необходимо сделать для минимизации этих рисков и проблем. Если бы у вас был выбор, то создавали бы вы гетерогенную среду или нет? Мотивируйте ваш ответ примерами.

Ответ.
-------------
Возможные проблемы использования гетерогенной среды виртуализации 
- рост стоимости обслуживания, выражаемый в необходимости обучения персонала работе с 
 различными гипервизорами
- необходимость лицензировать каждый гипервизор при использовании нескольких платных продуктов
- увеличение количества сопроводительной документации, описывающей все решения на гипервизорах

Для минимизации рисков и проблем всегда необходимо тщательно планировать инфраструктуру,
в том числе получая ответы в бизнес-процессах, возможное прогнозирование развития бизнеса, 
принятие упреждающих решений по модернизации и оптимизации решений
Если бы был выбор, создавать гетерогенную среду или нет, я бы действовал исходя из конкретной задачи, 
стараясь максимально выдержать баланс между сложностью системы (в сторну ее облегчения, читай - отказ от гетерогенности по возможности),
стоимостью ее содержания и максимально оптимальным расходованием ресурсов. т.е. если бы были
объективные причины отказаться от использования гетерогенной среды - я бы отказался:
нет смысла усложнять систему, если моногенная среда позволяет вместить пару неидеальных с т.з. производительности для 
гипервизора ОС. но если в используемом гипервизоре нет возможности развернуть ту или иную ОС, 
либо есть отрицательно значимые показатели использования той или иной ОС именно на текущем гипервизоре 
и использование другого решения позволит решить текущие проблемы, существенно повысив эффективность 
решения - лучше построить решение на базе гетерогенной среды виртуализации.
